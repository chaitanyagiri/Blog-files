Virtual DOM

So whats with the middle man i.e, virtual DOM why not we do things directly to DOM? The answer is simple time it saves us a lot of time if we introduce a virtual DOM.

DOM (Document Object Model) is a node tree that saves all the elements in a tree like structure and remakes the whole tree every time when you reload a page. While our friend Virtual DOM here it is also a node tree but it contains just the react components in it and it rebuilds everything under that component every time the component gets rendered. The virtual DOM is so fast that it can produce 200,000 nodes per second.

Rendering is the process where Virtual DOM under a component is rebuild due to state change etc. and the actual DOM is updated through an algorithm named diffing algorithm (Thats way too much to know for a beginner).

This is how messy it is and believe me I have barely scratched the surface while telling you this though it could be very interesting to know how these all work but it is beyond the scope of this tutorial series. From my perspective you should at least have the overview of what all is happening under the hood. These terms will be used very frequently in future articles and challenges so it was necessary to have an idea of what the heck are those.
